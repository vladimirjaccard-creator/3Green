# Directives

SOPs (Standard Operating Procedures) written in Markdown.

Each directive defines:
- **Goal** — What the task accomplishes
- **Inputs** — What data/parameters it needs
- **Execution** — Which script(s) to run and in what order
- **Outputs** — What gets produced (files, API calls, deliverables)
- **Edge Cases & Learnings** — Known gotchas, API limits, timing notes

## Naming Convention

Use descriptive, snake_case filenames:
```
scrape_website.md
generate_report.md
sync_google_sheets.md
```

## Template

Use `_template.md` as a starting point for new directives.
